# Figra 产品需求文档

## 1. 文档信息

- 产品名称：Figra
- 目标平台：macOS
- 当前产品形态：本地优先的桌面 PDF 工具箱
- 工程目录：`/Users/yangye/github/pdf2figure`
- App 工程：`FigraApp/`
- 运行依赖：随 app 打包的 `pdffigures2.jar` 和精简 JRE
- 文档用途：作为后续需求拆解、实现验收、发布说明和回归测试的产品基线

## 2. 产品定位

Figra 面向科研阅读、论文图表复用、报告整理、参考文献整理和日常 PDF 处理。它优先解决用户在本地快速处理论文 PDF 和文档 PDF 的任务，不做云端资料库或账号型产品。

核心价值：

- 本地处理：PDF、图片和 BibTeX 内容默认不上传。
- 低打扰工作流：每个工具聚焦一个任务，输出为新文件，不覆盖原文件。
- 论文友好：自动提取论文 Figure/Table，并支持高分辨率 PNG 复用。
- 日常可用：覆盖页面提取、整理、合并、拆分、压缩、隐私清理和加密等高频 PDF 操作。

## 3. 目标用户和场景

目标用户：

- 科研人员和学生：从论文中提取图表，整理阅读材料，复用图表到汇报或笔记。
- 报告和教学材料制作者：把 PDF 页面导出为图片或重新组合为新 PDF。
- 文档处理用户：合并、拆分、压缩、清理元数据或加密 PDF。
- 使用 BibTeX 的写作者：合并、去重和格式化 `.bib` 文件。

典型场景：

- 从一篇论文 PDF 中自动提取 Figure/Table，选择图片后复制到 Keynote、PowerPoint 或笔记应用。
- 从 PDF 中选择若干页面，以 600 DPI 导出 PNG 或另存为一个新 PDF。
- 删除 PDF 中不需要的页面，拖动调整顺序后导出整理版。
- 合并多个 PDF，或把长 PDF 按单页、选中页、每 N 页拆分。
- 分享 PDF 前清除 document info metadata，或给 PDF 设置打开密码。
- 合并多份 `.bib`，清理重复引用键或重复标题，并输出格式化文件。

## 4. 产品原则

- 所有处理默认在本地完成。
- 所有 PDF 操作默认生成新文件，不修改原 PDF。
- 正式 app 图表提取只使用内置 `pdffigures2`，不提取 PDF 内嵌对象。
- 图表提取输出使用 PNG；默认 DPI 为 `600`，固定档位为 `150 / 300 / 600 / 900`。
- 正式 app 使用随包 JRE，不依赖系统 Java、Python 或 conda。
- 论文 PDF、论文图片产物和 draw.io 复刻文件不进入工程目录，除非明确作为测试样例或资源。
- 发布 release 不手动创建或上传附件，只推送版本 tag，由 GitHub Actions 自动构建并发布唯一的 `Figra-vX.Y.Z.dmg` 和 `.sha256`。

## 5. 信息架构

主窗口使用原生 macOS 结构：左侧 source-list 工具导航，右侧展示当前工具工作区。

固定工具入口：

- 图表提取
- 页面提取
- 页面整理
- 合并 PDF
- BibTeX 工具
- 拆分 PDF
- 压缩 PDF
- 清除隐私
- PDF 加密
- 最近任务
- 输出设置

通用界面要求：

- 每个工具都有空状态、运行中、失败和完成状态。
- 右侧工作区保持单任务焦点，避免结果区挤压主操作区。
- 支持选择文件和拖入文件。
- 输出完成后可进入最近任务，并可打开或在 Finder 中定位。
- 默认浅色模式使用系统语义色和白底内容区，不使用大面积半透明灰遮罩。
- 操作按钮文案必须描述结果，例如“导出加密副本”“每页保存为 PDF”。

## 6. 通用输出规则

- 默认保存面板以输出设置中的目录作为初始目录。
- 输出文件不覆盖原文件。
- 当工具自动生成输出目录时，目录名应包含源文件名或任务名，若目录已存在则追加时间戳或使用保存面板避免覆盖。
- 图片导出使用当前 DPI 档位。
- 导出成功后记录最近任务，包含工具名、输出标题、简要详情、输出路径和时间。
- 开启“导出后自动定位输出”时，导出完成后在 Finder 中显示结果。

## 7. 功能需求

### 7.1 图表提取

用户目标：

- 选择或拖入一个论文 PDF。
- 使用内置 `pdffigures2` 自动提取 Figure/Table。
- 提取完成后看到摘要，并通过图库集中查看和复用图片。

功能需求：

- 每次只处理一个 PDF。
- 使用 `.app/Contents/Resources/jre/bin/java` 执行 `.app/Contents/Resources/pdffigures2.jar`。
- `pdffigures2.jar` 必须从 `FigraApp/Resources/pdffigures2.jar` 打包进 `.app/Contents/Resources/`。
- `FigraApp/Resources/jre/` 必须打包进 `.app/Contents/Resources/jre/`。
- 输出 PNG，DPI 使用当前选择档位，默认 600 DPI。
- 输出文件夹带 PDF 文件名前缀，例如 `{PDF文件名}_pdffigures2/`。
- 单张 PNG 文件名不重复携带 PDF 文件名前缀。
- 输出目录已存在时自动追加时间戳，避免覆盖。
- 提取失败时保留原始日志，供用户或开发者定位问题。
- 缺少 JRE 或 jar 时显示资源损坏提示。

验收标准：

- 内置 JRE 和 `pdffigures2.jar` 存在时能成功启动提取。
- 未识别到 Figure/Table 时显示空结果摘要，不崩溃。
- 成功提取后 `figures/` 下的 PNG 文件名不带 PDF 文件名前缀。
- DPI 切换后，重新提取的 PNG 尺寸随 DPI 变化。
- 原 PDF 不被修改。

### 7.2 图库面板

用户目标：

- 查看全部提取图片和表格。
- 快速复制图片或表格到剪贴板。
- 对选中结果执行打开、定位、复制、移除和导出清单操作。

功能需求：

- 使用弹出式图库，不挤压主工作区。
- 图库提供关闭按钮。
- 支持图片/表格筛选。
- 展示全部缩略图和选中项的大图预览。
- 点击缩略图默认复制图片到剪贴板。
- 提供“打开图片”“在 Finder 中显示”“复制图片”“从结果移除”“导出清单 CSV”。
- 缩略图加载失败不影响其他图片展示。

验收标准：

- 点击缩略图后可以粘贴到 Keynote、PowerPoint 或笔记应用。
- CSV 至少包含图片文件名和输出路径。
- 移除结果只影响当前结果列表，不删除磁盘文件。

### 7.3 页面提取

用户目标：

- 选择或拖入单个 PDF。
- 浏览全部页面缩略图和 PDF 大预览。
- 选择页面并导出图片或新 PDF。

功能需求：

- 显示全部页面缩略图。
- 支持点击、Command 多选、Shift 连续多选。
- 支持页码范围输入，例如 `1-3, 8, 10-12`。
- 缩略图选择与页码范围文本同步。
- 左侧缩略图选中时，右侧 PDF 预览同步跳转。
- 支持一键全选和清空选择。
- 支持复制单页为当前 DPI 的图片到剪贴板。
- 导出动作包括“保存为图片”“保存为一个 PDF”“每页保存为 PDF”“按连续分组保存 PDF”。
- 页码非法时显示错误，不执行导出。

验收标准：

- 载入 PDF 后显示全部页面缩略图。
- 选择 `1,3,5` 后导出图片，输出分辨率随 DPI 档位变化。
- 导出一个 PDF 时页面顺序符合选中顺序。
- 每页导出 PDF 时输出数量等于选中页数。
- 原 PDF 不被修改。

### 7.4 页面整理

用户目标：

- 从一个 PDF 中删除不需要的页面。
- 调整页面顺序后导出新 PDF。

功能需求：

- 支持选择或拖入单个 PDF。
- 显示页面列表和缩略图。
- 支持拖动重排和移除页面。
- 导出为新 PDF，不修改原 PDF。

验收标准：

- 导出页序符合当前列表顺序。
- 移除页面只影响导出副本。
- 原 PDF 不被修改。

### 7.5 合并 PDF

用户目标：

- 添加多个 PDF。
- 调整顺序后导出一个新 PDF。

功能需求：

- 至少两个 PDF 才允许导出。
- 列表每项显示文件名、页数和大小。
- 列表每项提供上移、下移和移除。
- 支持拖拽排序。
- 导出为新 PDF，不修改原文件。

验收标准：

- 两个及以上 PDF 可合并。
- 拖拽排序和上移/下移都生效。
- 导出的 PDF 页序符合列表顺序。

### 7.6 BibTeX 工具

用户目标：

- 添加一份或多份 `.bib` 文件。
- 按源文件优先级合并参考文献。
- 默认清理强重复项并格式化，导出前能预览最终 `.bib`。
- 对重复组逐组确认默认保留、改为保留其他条目或全部保留。

功能需求：

- 支持选择或拖入 `.bib` 文件。
- 首屏只保留添加和拖入入口；添加文件后左侧仍保留紧凑拖入入口。
- 左侧列表表达源文件优先级，显示 `1 / 2 / 3` 顺序，支持拖拽排序、上移、下移、移除和清空。
- 顶部提供“去重”和“格式化”两个开关，默认均开启，不再使用四任务选择器。
- 展示输入参考文献、重复组、将移除条目、最终输出数量。
- 中间工作区包含重复项、条目和输出三个视图；存在重复组时默认进入重复项审阅。
- 重复组显示保留项、候选项、移除项、匹配原因和关键字段对比。
- 重复组操作支持保留默认、改为保留任一候选、全部保留和恢复全部默认。
- 强重复项在去重开启时可自动移除，并在重复组中可见：DOI 相同、arXiv ID 相同、归一化标题相同、引用键和标题相同。
- 相同引用键但标题、年份或 DOI 不一致时视为 Key 冲突，默认全部保留并提示审阅。
- 疑似标题重复默认全部保留，只作为审阅提醒。
- 条目列表只高亮将移除、冲突、疑似和异常，不给普通保留项增加噪音徽标。
- 详情区优先展示字段摘要、重复理由和当前决策，BibTeX 原文可预览和复制。
- 格式化规则：统一条目缩进、字段换行和空行。
- 不修改原 `.bib` 文件。

验收标准：

- 单个 `.bib` 可格式化导出。
- 多个 `.bib` 可按列表顺序合并导出。
- DOI、arXiv ID、标题或引用键和标题相同的重复项会在默认设置下被清理，并显示统计。
- Key 冲突和疑似重复不会默认删除。
- 用户在重复组中切换保留项或全部保留后，输出预览和导出结果同步变化。
- 添加、移除或重排源文件后，重复组按新的优先级重新计算。
- `@comment`、`@preamble`、`@string` 等非普通引用条目不应被误当作可去重参考文献。
- 原 `.bib` 文件不被修改。

### 7.7 拆分 PDF

用户目标：

- 从长 PDF 中截取页面。
- 按单页或固定页数拆分。

功能需求：

- 复用页面提取的页面选择组件。
- 支持页码范围、缩略图多选、选中页导出为一个 PDF。
- 支持每页保存为单个 PDF。
- 支持每 N 页保存为一个 PDF。
- N 必须为正整数。

验收标准：

- 页码范围非法时显示错误，不导出。
- 每页拆分输出数量正确。
- 每 N 页拆分时，最后不足 N 页也导出。
- 原 PDF 不被修改。

### 7.8 压缩 PDF

用户目标：

- 为分享或归档生成体积优化副本。
- 理解当前压缩能力边界。

功能需求：

- 支持选择或拖入单个 PDF。
- 使用 PDFKit 重写结构生成优化副本。
- 显示导出前后的文件大小。
- 明确提示当前不重采样页面内图片，不承诺扫描件强压缩。

验收标准：

- 导出新 PDF，不覆盖原文件。
- 完成后显示体积变化。
- 对扫描图片类 PDF 不承诺明显压缩。

### 7.9 清除隐私

用户目标：

- 分享 PDF 前清理作者、标题、创建工具、时间等 document info metadata。
- 知道工具能力边界，避免误以为页面正文也被扫描。

功能需求：

- 支持选择或拖入单个 PDF。
- 显示 document info metadata 字段预览。
- 一键导出清理副本。
- 导出后可在 Finder 中定位清理副本。
- 不覆盖原 PDF。
- 明确提示只清除 PDF document info metadata，不承诺删除页面内容、批注或图片中的隐私信息。

验收标准：

- 清理副本不包含常见 document info 字段。
- 原 PDF 保持不变。
- 没有 metadata 时显示明确空状态。

### 7.10 PDF 加密

用户目标：

- 为 PDF 设置打开密码。
- 对已知密码的 PDF 导出无密码副本。

功能需求：

- 支持选择或拖入单个 PDF。
- 支持输入打开密码和可选所有者密码。
- 支持导出加密副本。
- 支持输入密码后导出无密码副本。
- 密码只在本次操作中使用，不保存。

验收标准：

- 原 PDF 不被修改。
- 加密副本再次打开时需要密码。
- 密码错误时显示明确失败状态。
- 解锁副本不继续要求打开密码。

### 7.11 最近任务

用户目标：

- 快速回到刚才导出的文件或文件夹。

功能需求：

- 记录本次运行中的导出历史。
- 历史记录包含工具、标题、详情、输出路径和时间。
- 支持打开输出文件或在 Finder 中定位。
- 不要求跨应用重启持久化。

验收标准：

- 完成导出后历史列表出现记录。
- 输出路径不存在时显示可理解的失败状态。

### 7.12 输出设置

用户目标：

- 减少重复选择输出目录和 DPI 的操作。

功能需求：

- 支持设置默认输出目录。
- 支持选择默认 DPI，固定为 `150 / 300 / 600 / 900`。
- 支持导出后自动定位输出。
- 设置影响之后的保存面板、图片导出和图表提取。

验收标准：

- 默认输出目录会作为保存面板初始目录。
- DPI 设置变更后，后续图片导出使用新 DPI。
- 自动定位开关打开后，导出完成会在 Finder 中显示结果。

## 8. 非功能需求

性能：

- 页面缩略图应后台生成，主界面保持可响应。
- 图表提取和 PDF 导出等耗时任务应显示运行状态。
- 大文件处理失败时应保留错误信息，不直接静默失败。

可靠性：

- 所有工具必须处理无法读取文件、空文件、加密文件、页码越界、输出目录不可写等错误。
- 后台任务完成后只更新当前有效任务状态，避免旧任务覆盖新选择文件的结果。

隐私和安全：

- 默认不上传文件。
- 不保存 PDF 密码。
- 最近任务只记录本次运行中的本地输出路径。

兼容性：

- App 目标平台为 macOS。
- 构建脚本使用 SwiftUI、AppKit、PDFKit 和 UniformTypeIdentifiers。
- 正式 app 不依赖系统 Java、Python 或 conda。

## 9. 明确不做

当前版本不做：

- 云端同步、账号系统或上传处理。
- OCR、公式识别、表格结构重建。
- 批量论文管理器。
- 图表提取的多 PDF 队列。
- PDF 内嵌对象拆解。
- 删除页面正文、图片、批注中可见的隐私内容。
- 在当前压缩工具中对扫描图片进行有损重采样。
- BibTeX 在线检索、引用元数据补全或 CSL 样式导出。

## 10. 发布和打包要求

- 构建 app 时复制 `FigraApp/Resources/pdffigures2.jar` 到 `.app/Contents/Resources/pdffigures2.jar`。
- 构建 app 时复制 `FigraApp/Resources/jre/` 到 `.app/Contents/Resources/jre/`。
- DMG 发布产物命名为 `Figra-vX.Y.Z.dmg`，并生成对应 `.sha256`。
- release 附件由 GitHub Actions 自动构建和发布。
- 不手动创建 release，不手动上传附件，避免重复或不一致产物。

## 11. 回归验收清单

发布前至少覆盖：

- 图表提取：普通论文 PDF、无 Figure/Table PDF、缺少 jar/JRE 的错误提示。
- 页面提取：范围选择、多选、复制页面、图片导出、单 PDF 导出、多 PDF 导出。
- 页面整理：重排、删除、导出页序。
- 合并 PDF：两个以上 PDF 排序合并。
- BibTeX 工具：合并、去重、格式化、去重格式化。
- 拆分 PDF：选中页、每页、每 N 页。
- 压缩 PDF：普通文本 PDF 和扫描图片 PDF 的提示与输出。
- 清除隐私：有 metadata 和无 metadata 的 PDF。
- PDF 加密：加密副本、错误密码、解锁副本。
- 最近任务和输出设置：默认输出目录、DPI、自动定位输出。
